Custom Commands

The Run Custom Command Dialog

This dialog interaction is for running emerge with options that are not yet integrated into Porthole. Some combinations of commands may never be fully integrated into porthole's GUI interface, and this allows you to run any custom emerge-related tool you may have lying about.

Caution! This dialog is capable of sending any command to porthole's terminal. Porthole's terminal does not handle all output formatting and is only intended to handle a limited range of commands.

The dialog opens with a blank command entry box and a drop down history selector. The first 6 entries in the history list are "persistent" ones that save you some typing. All you need to do is select one, fill in any missing info to complete the command. The remaining entries (if any) are the most recent history of commands actually run from the dialog. Porthole will remember up to 10 unique entries in total unless you edit your ~.porthole/prefs.xml file


Figure 1. Run Custom Command Dialog

Buttons

  • Entry box down arrow - displays the entry box history selector
  • Help - brings up this window
  • Cancel - closes window without executing an emerge
  • Execute - Uses the command string entered and submits it to the terminal window for execution.

Suggestions

Some useful commands to run (as long as you know what they do) are:

  • emerge --update --deep --newuse world (--pretend --verbose)
  • revdep-rebuild
  • emerge metadata

The following commands are known not to work reliably due to their output formatting:

  • etc-update
  • dispatch-conf
